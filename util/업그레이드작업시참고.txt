
DVSwitch 또는 dvsMU의 변경내용이 있을때 작업 방법 및 순서

2가지를 고려하여야 한다.
1. dvsMU에서 업그레이드하는 것
2. 새로운 dvs 이미지를 만드는 것


공통작업
1. 기존의 SD카드에 설치된 내용에, 변경할 파일을 수정하고 테스트한다.
2. 테스트가 완료되면, 변경된 파일을 컴퓨터에 다운로드한다.


dvsMU에서 업그레이할때를 고려한 작업
1. 파일을 HL5BTF의 github에 올린다.
2. github의 upgrade.sh파일을 수정하여 업그레이드시 처리할 내용을 추가한다.


새로운 dvs 이미지 만드는 방법
1. 직전의 dvs 이미지를 다운로드 하여 SD카드에 설치한다. https://drive.google.com/drive/u/0/folders/1ORv7ehiaA3olCU5YMU7j8d4y7i2xTj08
2. 다운로드 한 파일을 SD카드로 업로드한다. 또는 github에서 다운로드한다.
    dvsMU의 경우는 이렇게 다운로드한다. sudo wget -O /usr/local/dvs/dvsmu https://raw.githubusercontent.com/hl5btf/DVSMU/main/dvsmu
3. DVSwitch가 업그레이드되었다면, DVSwitch도 업그레이드한다. sudo apt-get update     sudo apt-get upgrade -y
4. SD카드의 내용을 그대로 이미지를 만든다.
5. Google Drive에 올린다

이미지작업후 필히 확인할 내용 (부팅시 /etc/dvsstart.sh가 작동하여 특정 파일의 값들이 바뀔수 있음)
1. var.txt에 startup_lan과 first_time_instl 이 73으로 되어 있지 않은지 확인
2. var.txt의 항목들의 값이 들어 있지 않은지 확인 (주파수는 그대로 둔다 430000000)
3. /etc/rc.local에 dvsstart.sh를 시작하는 라인이 있는지 확인 sudo /etc/dvsstart.sh &
4. /boot/dvsconfig.txt에 chg가 73으로 되어 있지 않은지 확인
5. /opt/MMDVM_Bridge/MMDVM_Bridge.ini 의 항목들의 값이 들어 있지 않은지 확인ㅊㅇ /ㄷㅅㅊ

2. github의 upgrade.sh파일을 수정하여 업그레이드시 처리할 내용을 추가한다.
